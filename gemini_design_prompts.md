# Gemini 디자인 요청 프롬프트 템플릿 (2048 이미지 블록 앱)

## 사용 방법
- 아래 템플릿을 그대로 복사해 Gemini에 전달한다.
- `[ ]` 구간만 현재 스프린트 정보로 교체한다.
- 요청 시 항상 `밸런스/재미요소/이펙트` 항목을 함께 포함한다.

## 0) 공통 베이스 프롬프트
```text
모바일 2D 게임 UI/아트 스타일로 2048 퍼즐 앱의 화면 디자인을 제작해줘.
숫자 타일 대신 단계별 진화 이미지 블록을 사용한다.
아래 조건을 반드시 반영:
1) 화면 비율: 1080x1920 (세로)
2) UI 안전영역 고려 (상단 노치/하단 배너 광고 영역)
3) 하단 배너 광고 영역은 320x50 또는 360x60 기준으로 공간 확보
4) 직관적인 터치 인터페이스와 높은 가독성
5) 각 화면은 동일한 세계관/색감/아이콘 체계 유지
6) 합성 성공 시 강한 시각적 피드백(글로우, 파티클, 잔상)
7) 결과물: 메인 시안 1개 + 컬러 변형안 2개

현재 테마:
- 세계관: [예: 숲의 성장, 동물 진화, 보석 합성]
- 키 컬러: [예: 민트 + 골드 + 다크네이비]
- 분위기: [예: 캐주얼, 경쾌, 중독성 있는 아케이드]

밸런스/재미 요소 반영:
- 플레이 초반은 안정적, 후반은 긴장감 상승이 느껴지는 UI 연출
- 콤보 단계가 올라갈수록 이펙트 강도 증가
- 목표 단계 직전에는 긴장감을 주는 강조 연출 추가
```

## 1) 프로그램 실행 화면 (Splash/Loading)
```text
[공통 베이스 프롬프트]를 기반으로 프로그램 실행 화면을 디자인해줘.
필수 요소:
- 게임 로고
- 짧은 로딩 인디케이터
- 세계관을 암시하는 배경 오브젝트
- 시작 2초 내 시선 집중 가능한 중앙 포인트

연출 요구:
- 로고 등장 시 부드러운 스케일 업
- 로딩 중 은은한 파티클 움직임

산출물:
- 스플래시 화면 시안 1개
- 로딩 포함 버전 1개
```

## 2) 로비 화면 (Lobby/Home)
```text
[공통 베이스 프롬프트]를 기반으로 로비 화면을 디자인해줘.
필수 요소:
- 플레이 시작 버튼(가장 강조)
- 포인트 잔액 표시
- 아이템 상점 진입 버튼
- 일일 미션/랭킹/설정 아이콘
- 하단 배너 광고 고정 영역

기획 반영:
- 유저가 3초 이내에 "시작"과 "보상/상점"을 인식 가능해야 함
- 과금 유도 느낌보다 성취 보상 중심 톤

산출물:
- 로비 기본형 1개
- 이벤트 배너 확장형 1개
```

## 3) 메인 게임 화면 (2048 Image Board)
```text
[공통 베이스 프롬프트]를 기반으로 메인 게임 화면을 디자인해줘.
필수 요소:
- 4x4 보드
- 이미지 블록 단계별 구분이 명확한 스타일
- 현재 점수, 최고 점수, 콤보 게이지
- 아이템 버튼 5종(되돌리기/셔플/해머/더블스코어/스타트업)
- 하단 광고 영역과 충돌하지 않는 입력 구조

이펙트 요구:
- 합성 시: 글로우 + 파티클 + 짧은 잔상
- 고단계 합성 시: 화면 외곽 강조 연출
- 연속 콤보 시: 색상 온도 상승

밸런스 체감 요구:
- 초반은 시각적으로 안정적
- 후반으로 갈수록 긴장감 있는 대비/연출 증가

산출물:
- 인게임 기본형 1개
- 고콤보 상태 연출형 1개
```

## 4) 미니게임 화면 (Bonus Mode)
```text
[공통 베이스 프롬프트]를 기반으로 미니게임 화면을 디자인해줘.
목표:
- 30~60초 짧은 플레이에서 보너스 포인트 획득 감각 제공

필수 요소:
- 남은 시간 타이머
- 미니게임 점수
- 즉시 재도전 버튼
- 본 게임 복귀 버튼

연출 요구:
- 속도감 있는 UI 모션
- 성공/실패 피드백이 즉시 전달되는 컬러 시스템

산출물:
- 미니게임 기본형 1개
- 하이텐션 이벤트형 1개
```

## 5) 결과/보상/상점 화면 (Result + Reward + Shop)
```text
[공통 베이스 프롬프트]를 기반으로 결과/보상/상점 화면을 디자인해줘.
필수 요소:
- 최종 점수/최고 기록/획득 포인트
- 클리어 보상 수령 버튼
- 아이템 구매 리스트
- 다음 판 시작 버튼
- 하단 배너 광고 영역

경제 시스템 반영:
- 아이템 가격 가시성 명확
- 현재 포인트로 구매 가능한 항목을 우선 강조
- 과도한 정보 밀도 방지

산출물:
- 결과 중심형 1개
- 상점 중심형 1개
```

## 6) 이미지 블록 에셋 요청 프롬프트
```text
2048 숫자 단계(2~2048)를 이미지 진화 단계로 치환한 타일 세트를 만들어줘.
조건:
- 단계가 올라갈수록 희귀도와 광택이 증가
- 작은 크기(64~128px)에서도 식별 가능
- 배경에서 떠보이는 외곽선/그림자 처리
- 동일한 프레임/여백 규칙 유지

요청 출력:
- 단계별 키아트 가이드
- 아이콘 스타일 가이드
- 합성 전/후 상태 비교 시안
```

## 7) 이펙트/피드백 전용 프롬프트
```text
다음 이벤트별 2D 이펙트 스타일 프레임을 제안해줘:
1) 일반 합성 성공
2) 고단계 합성 성공
3) 연속 콤보 달성
4) 목표 단계 직전 경고
5) 클리어 연출

조건:
- 저사양 모바일에서도 부담이 적은 표현
- 과한 화면 점멸 금지
- 색약 유저도 상태를 구분 가능한 대비 유지
```

## 8) 한 번에 요청할 통합 프롬프트 (실무용)
```text
2048 이미지 블록 모바일 게임의 전체 UI 스타일팩을 제작해줘.
필요 화면:
1) 프로그램 실행 화면
2) 로비 화면
3) 메인 게임 화면
4) 미니게임 화면
5) 결과/보상/상점 화면

핵심 기획:
- 하단 배너 광고 영역 고정 고려
- 포인트로 아이템을 구매해 고득점을 지원
- 콤보/고단계 합성에서 강한 성취감 이펙트 제공
- 초반 접근성 + 후반 긴장감이 공존하는 시각적 구조

결과물 형식:
- 화면별 와이어 + 스타일 시안
- 컬러/타이포/아이콘 시스템
- 블록 단계별 아트 가이드
- 이펙트 가이드(합성/콤보/클리어)
```
