# 2048 이미지 블록 앱 개발용 AGENTS

## 1) 프로젝트 목표
- 숫자 타일 2048을 `이미지 블록` 기반 모바일 앱으로 재구성한다.
- 역할 분업형 운영으로 기획, 밸런스, 개발, 테스트, 디자인 요청을 유기적으로 연결한다.
- 하단 배너 광고와 포인트-아이템 경제를 포함해 실제 서비스 가능한 형태로 완성한다.

## 2) 에이전트(업무 분업) 구조

### A. 총괄 기획 에이전트 (PM/Game Design)
- 게임 코어 규칙, 화면 흐름, 컨텐츠 우선순위 정의
- 포인트 경제와 아이템 사용 정책 수립
- 재미요소(콤보, 연출 타이밍, 미니게임 보상 루프) 설계
- 산출물: GDD 요약, 스프린트 백로그, 기능 우선순위

### B. 밸런스 에이전트 (Balance/Economy)
- 블록 생성 확률, 점수 곡선, 난이도 상승 구간 설계
- 아이템 가격/효율/쿨다운 계산 및 악용 방지 룰 설계
- 포인트 수급량(클리어/실패/미니게임) 최적화
- 산출물: 밸런스 표, 시뮬레이션 로그, 튜닝 변경 이력

### C. 클라이언트 개발 에이전트 (Game/App Dev)
- 화면/상태/입력/애니메이션/광고 SDK 통합 구현
- 이미지 블록 합성, 점수 계산, 저장/복구, 튜토리얼 구현
- 성능 최적화(프레임, 메모리, 로딩)
- 산출물: 기능 코드, 빌드, 릴리즈 노트

### D. QA 테스트 에이전트 (QA/Automation)
- 기능 테스트, 회귀 테스트, 광고/결제/포인트 검증
- 밸런스 체감 테스트(초보/숙련 페르소나) 수행
- 버그 재현 문서화 및 수정 검증
- 산출물: 테스트 케이스, 결함 리포트, 릴리즈 체크리스트

### E. 라이브 운영 에이전트 (LiveOps/Data)
- DAU, 세션 길이, 클리어율, 아이템 사용률, 광고 노출률 분석
- A/B 테스트(아이템 가격, 보상량, 이펙트 강도) 제안
- 주간 이벤트와 보상 정책 운영
- 산출물: 대시보드 지표, 운영 리포트, 실험 결과

### F. 디자인 협업 에이전트 (Art Direction + Gemini Prompt)
- 화면별 UI/배경/블록/아이콘/이펙트 스타일 가이드 정의
- Gemini 요청 프롬프트 생성 및 수정 피드백 반영
- 기획/밸런스 변경사항을 디자인 요청문에 즉시 동기화
- 프롬프트 작성 시 **법적/저작권 안전 규칙**을 반드시 포함
- 산출물: 프롬프트 문서, 스타일 가이드, 에셋 체크리스트

## 2-1) 프롬프트 법적 안전 규칙 (필수)
프롬프트 파일을 만들 때 아래 규칙을 반드시 포함하고, 생성 결과도 동일 기준으로 검수한다.

- 특정 IP/브랜드/회사/게임/캐릭터/인물/작가/스튜디오 명칭 사용 금지
- 로고/상표/유명 캐릭터/저작권 보호 요소가 연상되는 형태 금지
- “~스타일로”, “~같이”, “~풍”처럼 특정 기존 작품을 모방하는 지시 금지
- 텍스트가 필요 없으면 텍스트 삽입 금지 (본 프로젝트는 기본 텍스트 금지)
- 결과물에 유사 요소가 보이면 즉시 재요청(재생성) 처리

프롬프트에 항상 포함할 안전 문구(복붙):
```text
법적 안전 규칙:
- 특정 IP/브랜드/로고/캐릭터/인물/작가/스튜디오를 연상시키는 요소는 금지.
- 기존 작품의 스타일/구도를 모방하지 말고, 완전히 독창적인 결과물로 제작해줘.
```

## 3) 화면 분류 (5개 권장)

### Screen 1. 프로그램 실행 화면 (Splash/Loading)
- 앱 로고, 세계관 키비주얼, 로딩 인디케이터
- 초기 데이터 로드, 버전 체크, 로그인 토큰 검사
- 전환: 완료 시 로비 화면으로 이동

### Screen 2. 로비 화면 (Lobby/Home)
- 시작 버튼, 랭킹, 아이템 상점, 일일 미션, 설정 진입
- 포인트 잔액/재화 상태 표시
- 하단 배너 광고 고정 노출 위치 1순위

### Screen 3. 메인 게임 화면 (2048 Image Board)
- 4x4 보드, 스와이프 입력, 이미지 블록 합성, 점수/콤보 UI
- 아이템 버튼(되돌리기, 셔플, 해머 등)과 잔량 표시
- 게임 종료/클리어 시 결과 화면 또는 보상 흐름으로 이동

### Screen 4. 미니게임 화면 (Bonus Mode)
- 짧은 플레이(30~60초)로 포인트 보너스 획득
- 본 게임 대비 단순 규칙으로 재도전 유도
- 일일 횟수 제한/광고 보상 연계 가능

### Screen 5. 결과/보상/상점 화면 (Result + Reward + Shop)
- 판 종료 점수, 최고 기록, 획득 포인트 표시
- 클리어 보상 수령, 아이템 구매/강화, 다음 플레이 동선 제공
- 광고 보상형 버튼(선택 시 추가 포인트) 배치 가능

## 4) 이미지 블록 기획 (숫자 대체)
- 기존 숫자 단계(2,4,8...)는 `진화형 이미지` 단계로 매핑한다.
- 예시 테마:
  - 2: 씨앗
  - 4: 새싹
  - 8: 잎
  - 16: 꽃봉오리
  - ...
  - 2048: 완성체(레전드 이미지)
- 합성 시 다음 단계 이미지로 자연스럽게 변환되도록 파티클/글로우 연출을 동기화한다.

## 5) 포인트/아이템 경제 기획

### 포인트 획득
- 기본 점수 기반 포인트 전환: `포인트 = floor(게임점수 / 120)`
- 클리어 보너스: +150 포인트
- 연속 클리어 보너스: +30% (최대 3연속)
- 미니게임 보상: 50~120 포인트 범위

### 아이템 (고득점 보조)
1. 되돌리기 토큰
- 효과: 최근 1수 취소
- 가격: 120 포인트
- 제한: 게임당 3회

2. 셔플 토큰
- 효과: 현재 블록 배치만 재섞기
- 가격: 180 포인트
- 제한: 게임당 2회

3. 해머 토큰
- 효과: 선택 블록 1개 제거
- 가격: 260 포인트
- 제한: 게임당 1회

4. 더블 스코어 부스트
- 효과: 20초간 점수 2배
- 가격: 220 포인트
- 제한: 게임당 2회

5. 스타트 업그레이드
- 효과: 시작 블록을 1단계 상향
- 가격: 300 포인트
- 제한: 게임 시작 전 1회

### 밸런스 가드레일
- 아이템 과사용 시 점수 보정치 적용(랭킹 분리 또는 보정계수)
- 무료 재화 수급량은 일일 상한 적용
- 초보 구간은 체감 난이도 완화, 상위 점수 구간은 효율 점진 하락

## 6) 재미요소/이펙트 기획 기준
- 콤보 단계별 카메라 흔들림 강도 증가
- 고단계 합성 시 화면 외곽 빛 번짐 + 짧은 슬로모션
- 연속 합성 체인 시 사운드 피치 업
- 클리어 직전(목표 단계 1단계 전) 긴장 연출 UI 표시
- 미니게임 성공 시 메인 게임 보너스 버프 부여

## 7) 광고 정책 (하단 배너 중심)
- 기본 원칙: 플레이 몰입 방해 최소화
- 배너 위치: 로비/결과/상점 화면 하단 고정
- 메인 게임 화면은 선택 적용:
  - 방법 A: 안전영역 하단 고정(입력 영역 비침범)
  - 방법 B: 일시정지/종료 직후만 노출
- 클릭 오작동 방지 여백 및 버튼 간격 규칙 유지

## 8) 협업 워크플로우 (유기적 대응)
1. 기획 에이전트가 주간 목표와 변경 요구 정의
2. 밸런스 에이전트가 수치 시뮬레이션 및 제안서 업데이트
3. 디자인 협업 에이전트가 Gemini 프롬프트에 변경사항 반영
4. 개발 에이전트가 기능 구현 및 이벤트/이펙트 연동
5. QA가 기능/밸런스/광고 검증 후 이슈 리포트
6. 라이브 운영 에이전트가 지표 기반 재조정 제안

## 9) 완료 기준 (Definition of Done)
- 5개 화면 기본 플로우가 끊김 없이 동작
- 이미지 블록 합성, 점수, 포인트, 아이템 로직 정상 동작
- 하단 배너 광고 위치/오작동 이슈 없음
- 기본 테스트 케이스 통과율 95% 이상
- Gemini 디자인 에셋 반영 후 UI 일관성 체크 완료

## 10) 디자인 요청 문서
- 화면/오브젝트 디자인 요청은 `gemini_design_prompts.md`를 기준으로 사용한다.
